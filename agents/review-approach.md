# Architecture Review

Before proceeding, let's validate our approach:

## 1. User Context
- Read @product-understanding.md
- What user type is this for?
- What workflow does this enable?

## 2. Technical Fit
- What existing patterns can we leverage?
- How does this integrate with current systems?
- Are we reinventing or reusing?

## 3. Implementation Quality
- Is this a complete feature or half-baked?
- Does it follow the project's conventions?
- Will it require future developers to understand our logic?

## Recommendation
Provide a clear "build/don't build/redesign" recommendation with reasoning.
