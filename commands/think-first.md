# Think First Protocol

STOP! Before writing ANY code for: $ARGUMENTS

## Required Analysis (use the solution-architect agent)

1. What problem are we REALLY solving?
2. What's the complete user journey?
3. What already exists we can use?
4. How will components connect?

**Prior context (if the mnemosyne plugin is installed):** search past sessions and memories for this feature area (`search_memories` / `search_sessions` MCP tools) - earlier decisions about the same area count as "existing solutions". Skip silently if unavailable.

## Only After Analysis

- Get the architect's blueprint
- Validate the approach
- THEN start implementation

## Remember

We're building bridges, not islands.
Code that doesn't connect is technical debt.
